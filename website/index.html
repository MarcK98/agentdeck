Features

Everything a one-person AI team needs.

Ticket board

A kanban cockpit where live agent runs render as pulse-dot cards. Write a ticket, hit delegate, and an agent spins up on its own git branch.

Always-on team lead

A lead agent wakes on a heartbeat, reads the board, delegates coding to project threads, and pings you only when it needs a decision.

Live threads

Every run is a thread: an activity trail of tools and turns, a steering composer to redirect mid-flight, and a context rail with cost, branch, and PR status.

Approvals inbox

Agents never block waiting on you. Permission prompts land in a non-blocking inbox — full command JSON, allow once or deny, decided trail kept.

Usage, hour by hour

Token and dollar tracking across 1h to 30-day windows: by model, by project, per live session with context bars. Know your burn before the invoice does.

Worktree isolation + PR loop

Each ticket gets its own git worktree and branch. Agents open PRs, an automated reviewer responds, change requests re-run the author — you merge.

Mobile companion

Board, approvals, and live runs from your phone. The daemon dials out to a relay — your machine never opens an inbound port.

Native notifications

A comment lands on a ticket or an agent needs approval — you get an OS notification that clicks through to the exact thread.

MCP + connectors

Per-project MCP servers with secret storage, plus first-class Google Cloud and App Store Connect auth. Your agents get your tools.

How it works

One daemon. Every screen is a client.

Spawn is a local-first system: a background daemon owns your Claude Code sessions, tickets, approvals, and usage in a SQLite store on your machine. Everything else just talks to it.

Daemon

Owns sessions, threads, tickets, approvals, usage. SQLite on disk. Your code and keys never leave the machine it runs on.

Desktop

Electron Mission Control: board, threads, approvals, usage, live agent map. Auto-starts the daemon.

Relay → Mobile

The daemon dials out to a stateless relay; your phone dials in. No inbound ports, no VPN, no port-forwarding.

Open source

Yours to run. No lock-in, ever.

The entire product is public — daemon, desktop app, relay, mobile. Bring your own Claude Code subscription, clone the repo, and run the whole thing yourself for free. Paid plans exist for one reason: you'd rather we operate it, keep it updated, and answer the pager.

FAQ

Questions, answered.

Is self-hosting really the full product?

Yes. There is one codebase and it's public. Cloud runs the exact same daemon and relay you can run yourself — you're paying for operations, updates, and support, not unlocked features.

Where do my code and API keys live?

Self-hosted: entirely on your machine — the daemon stores everything in a local SQLite database, and the relay only forwards encrypted traffic your daemon initiates. Cloud: the managed daemon runs in an isolated environment per customer; your Claude credentials are yours and usage is billed by Anthropic to you directly.

Do I need a Claude subscription?

Yes — Spawn orchestrates Claude Code, it doesn't resell it. A Claude Code subscription or an Anthropic API key is required on every plan. Spawn's usage view helps you see exactly what you're spending.

Why Claude only?

By design. Spawn is built against Claude Code's real capabilities — sessions, skills, MCP, subagents, worktrees — instead of a lowest-common-denominator abstraction over five providers. Depth over breadth.

What does "constant updates" mean on Cloud?

Spawn ships continuously — the changelog is public. On Cloud we roll every release out to your instance for you, with backups taken before each update. Self-hosters pull whenever they like.

Can I use it from my phone?

Yes. The mobile companion shows your board, approvals, and live runs, and lets you steer threads. It connects through the relay, so your desktop never exposes an inbound port.
